# Skills – DataSync Content Store MongoDB

Source of truth for detailed guidance. Read [`AGENTS.md`](../AGENTS.md) first, then open the skill that matches your task.

## When to use which skill

| Skill folder | Use when |
| --- | --- |
| `dev-workflow` | Branching, running build/test/lint, CI, releases |
| `mongodb-content-store` | Public API, DataSync wiring, MongoDB config and behavior |
| `typescript` | Editing `src/**/*.ts`, `tsconfig.json`, or TSLint |
| `testing` | Adding or changing Jest tests, mocks, or coverage |
| `code-review` | Preparing or reviewing a PR for this package |

Each folder contains `SKILL.md` with YAML frontmatter (`name`, `description`).
